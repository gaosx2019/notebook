1-2 常用算法的平均时间复杂度

常用算法的平均时间复杂度计算​

冒泡排序​

//冒泡排序
void buddle_sort(int *p,int n)
{
        int i = 0,j = 0;        

        for(i = 0;i < n;i++) //轮数
        {
                for(j = 0;j < n - 1 - i;j++)
                {
                        if(p[j] > p[j + 1])        
                        {
                                p[j] ^= p[j + 1];        
                                p[j + 1] ^= p[j];        
                                p[j] ^= p[j + 1];        
                        }
                }
        }
        return ;
}

说明:我们主要计算核心算法的平均时间复杂度。核心语句为内部的if语句。

i = 0， 内部if语句执行了 n - 1次
i = 1， 内部if语句执行了 n - 2次
i = 2， 内部if语句执行了 n - 3次
..
i = n - 2,内部if语句执行了1次

核心语句执行的频度:  T(n) = 1 + 2 + 3 + .... + n -1 
                        = n * (n - 1) / 2
对T(n)求极限(可以立即成最高次幂)可得到
冒泡排序的时间复杂度:  O(n^2)

简单选择排序​

//简单选择排序
void select_sort(int *p,int n)
{
        int i = 0,j = 0,k = 0;

        for(i = 0;i < n - 1;i++) //遍历数组中除最后一个元素外所有的元素
        {
                k = i; //k记录起始值
                for(j = k + 1;j < n;j++)
                {
                        if(p[j] < p[k])        
                                k = j;
                }

                if(k != i)
                {
                        p[i] ^= p[k];        
                        p[k] ^= p[i];        
                        p[i] ^= p[k];        
                }
        
        }
        return ;
}

说明:我们主要计算核心算法的平均时间复杂度。核心语句为内部的if(p[j] < p[k])语句。

i = 0， 内部if语句执行了 n - 1次
i = 1,  内部if语句执行了 n - 2次
i = 2,  内部if语句执行了 n - 3次
i = n - 2,内部if语句执行了1次.

核心语句执行的频度:  T(n) = 1 + 2 + 3 + .... + n -1 
                        = n * (n - 1) / 2
对T(n)求极限可得到
简单选择排序的时间复杂度:  O(n^2)

快速排序​

void quick_sort(int a[],int low,int high)
{
        int i = 0,j = 0;
        int key = 0;
        i = low;  //保存第一个有效元素的下标
        j = high; //保存最后一个有效元素的下标

        key = a[low];

      while(i < j)
    {
        while(i < j && a[j] >= key )
                j--;
    
        if(i < j)
        {
                a[i] = a[j];
        }
    
        while(i < j && a[i] <= key)
                i++;
    
        if(i < j)
        {
            a[j] = a[i];        
        }

    }
        //填充key的值到a[i]的值
        a[i] = key;

        if(j - 1 > low)
        {
                quick_sort(a,low,j - 1);        
        }

        if(i + 1 < high)
        {
                quick_sort(a,i + 1,high);        
        }

        return ;
}

注： 快速排序由于存在递归，故不好有普通方法计算。需要用数学公式来进行计算。

最优情况： 每次取到的元素都刚好平分分整个数组。
此时时间复杂度的频度: T[n] = 2T[n/2] + f(n).

f(n)为此次平分数组所花费的时间复杂度。 【找出中值位置，即10行-28行的代码】
T[n/2]为平分后子数组的时间复杂度。 【即32-40行的代码】
在最优的情况下: f(n) = n;  最优情况下快排的时间复杂度如下:

第一次递归，令 n = n , T[n] = 2T[n/2] + n          
第二次递归，令 n = n/2,  T[n]= 2{2T[n/4] + (n / 2)} + n   【T[n/2]套用上面公式】
                   ===>T[n] = 2^2 T[n / 2^2] + 2n
第三次递归，令 n = n /(2^2), T[n] = 2^2 {2T[n/2^3] + n / 2^2} + 2n
                   ===>T[n] = 2^3T[n/2^3] + 3n;
....
以此类推
第k次递归(k次后结束) ,令n = n / （2^(k - 1)), T[n] = 2^k T[n/2^k] + kn
当最后不能在平分的时候，公式一直向下跌倒，最后只剩T[1]。公式迭代完成。(T[1]是常量)

T[n/2^k] = T[1] ===>  n / 2^k = 1  =====> n = 2 ^ k  ===> k = log2^n
故：
T[n] = nT[1] + nlog2^n

所以O(n) = O(nlog2^n)

最差的时间复杂度

最差情况：i 一次不执行，全部由j来执行。
T(n) = T(0) + T(n - 1) + f(n)
     =  T(n - 1) + n;
第一次递归，令 n = n ,          T[n] = T(n - 1) + n
第二次递归，令 n = n - 1,       T[n] = T(n - 2） + n - 1 + n
第三次递归，令 n = n - 2,       T[n] = T(n - 3） + n - 2 + n - 1 + n
....

T(n) = 1 + 2 + 3 + 4 + ...n - 1 + n
     = (1 + n) * n / 2
     
时间复杂度: O(n^2)

平均时间复杂度

平均时间复杂度: O(nlog2^n)
